关于device-owner你所需要了解的一切

关于device-owner你所需要了解的一切

device-owner简介

在android中device-owner是设备的管理者，一旦一个app成为device-owner以后，将拥有如下权限:

  • 写settings数据库
  • 关闭gps
  • 静默安装、卸载应用（6.0以上）
  • 防卸载
  • 冻结手机上的其他app etc

更多的功能可以看DevicePolicyManagerService的源码。

一个app如何才能成为device-owner？

1.有root权限

1）创建 device_owner.xml，内容如下：

2) adb push device_owner.xml /sdcard/

3)adb shell

4)su

5)cp /sdcard/device_owner.xml /data/system/

6)cd /data/system/

7)chown system:system device_owner.xml

8)reboot

2.无root权限

2.1 adb shell命令

在手机上不存在任何账户的情况下，可以通过adb命令：

adb shell dpm set-device-owner your.package.name/your.adminreceiver

将你的应用设置为device-owner。其中your.adminreceiver是DeviceAdminReceiver的子类，需要你自己实现。

ps：这种情况下手机上不能存在账户，可以通过如下方法来解决手机上存在的账户问题：

1）adb shell dumpsys account

查看该命令的输出结果，如果手机上存在账户的话，会看到Accounts字段不为0

2）adb shell pm hide 来禁用提供上面账号的应用然后在执行设置device-owner的命令

2.2 nfc方式

这种方式要求比较苛刻，需要两部手机，且这两部手机都需要有nfc功能，并且想要设置device-owner的那台手机需要先恢复出厂设置，因此在实际项目中用到的较少。
